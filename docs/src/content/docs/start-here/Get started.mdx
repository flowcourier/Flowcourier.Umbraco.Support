# How to install and use Google Analytics for Umbraco

Four steps get the extension up and running in Umbraco.

<Steps>

1. Install the extension

    The extension is available via NuGet. Visit [Google Analytics by Flowcourier on NuGet](https://www.nuget.org/packages/Flowcourier.Umbraco.Analytics/), search for the package in Visual Studio, or run:
    <Code code="dotnet add package Flowcourier.Umbraco.Analytics" lang="shell"  />

2. Create a service account for your Google Analytics property

    The extension reads your data through Google's APIs using a **service account**. Create one in the Google Cloud Console, then give it **Viewer** access to the Google Analytics property you want to report on — the same way you would invite any user.

    For detailed steps, see [2. Create Service Account](/docs/install-guide/2-create-sa-and-add-api-access) and [3. Add Service Account User to Google Analytics](/docs/install-guide/3-add-sa-user-to-google-analytics) in the Install Guide.

3. Get and install a license (optional for the Free Edition)

    Without a license the extension runs in **Free Edition** mode, with a 30-day data view limit. To unlock the paid features, purchase a license on the [Google Analytics for Umbraco page](https://flowcourier.com/solutions/google-analytics/) — we email the license file to you, and you copy it into your site's `/umbraco/Licenses` folder.

    The license is bound to the domains configured in Google Analytics, so when ordering, include every domain you want the extension to work on.

4. Create a connection to Google Analytics in Umbraco

    Once the package is installed and the service account has access to your property, create a new connection under **Settings → Flowcourier** in the Umbraco backoffice.

    For detailed steps, see [4. Extension configuration in Umbraco](/docs/install-guide/4-umbraco-configuration) in the Install Guide.

</Steps>
